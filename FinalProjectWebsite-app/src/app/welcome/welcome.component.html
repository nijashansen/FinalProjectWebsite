How does it work?

The process in blue prism has a HTTP Post method on the process when an error occurs, sending the error to the rest api, which saves it in the database. The Angular client makes a HTTP request of the type "GET" which gets the errors and process names. This makes it easier for the RPA team in Norlys to see processes which has failed and what kind of error it is.
